Two rounds of changes pushed to the claude/review-project-angus-Woph2 branch. Quick summary:

Round 1 — Scout agent (commit b32b081)

  • “No website” filter now ignores Facebook / Yelp / Linktree / business.site / OpenTable and similar social + aggregator URLs, so those listings still qualify as leads instead of being disqualified.
  • API results are category-verified against the requested category with synonyms (cafe → restaurant, barber → salon, yoga → gym), so text search can’t slip in off-category listings.

Round 2 — Efficiency pass (commit 077a5a9)

  • DB indexes on the hot query paths (businesses by zip/status, emails by status/sent_at, business_costs, token_usage, agent_logs, issues).
  • Real token accounting: Builder returns Claude’s response.usage + model; Accountant and Pricer now cost input / output / cache-read / cache-write separately per model instead of one flat $0.003/1K. The fake 150 / 200 / 500 / 2000 token constants in the pipeline are gone.
  • Prompt caching hint added to Builder’s system prompt (cache_control: ephemeral). It’s a no-op until the system prompt crosses Haiku’s 2048-token threshold — once we enrich it with style guidance or few-shots, caching activates with no further code changes.
  • Bounded parallelism: the pipeline now runs three leads concurrently through a worker pool matching the Builder count, with per-lead error isolation. Daily-email-limit SQL is hoisted out of the inner loop.
